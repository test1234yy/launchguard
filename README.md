# LaunchGuard

**Deployment readiness scanner for Next.js and Node.js projects.**

LaunchGuard scans a public GitHub repository or an uploaded ZIP for the mistakes that break real deploys — committed secrets, undocumented environment variables, unsafe dependencies, missing CI, Docker pitfalls, Prisma migration problems and configuration errors — then gives you a **0–100 readiness score** and a prioritized, copy-pasteable fix plan.

It is built with **Next.js 14 (App Router), React 18 and TypeScript**, and it runs entirely without a database, authentication or any paid service. A one-click **demo scan needs no API key and no network access**.

---

## Highlights

- 🔍 **58 deterministic rules** across 9 categories (secrets, environment, dependencies, CI, Docker, Prisma, Next.js, configuration, quality).
- 🎯 **Readiness score 0–100** with a human grade (“Launch ready” → “Not ready”), plus **projected scores** (“fix all criticals → 74”).
- 🧾 **Findings** carry severity, category, file (and line), redacted evidence, concrete remediation and an **effort estimate** (quick fix / moderate / involved).
- ⚡ **Quick wins** — the low-effort, high-impact fixes to do first.
- 🎛️ **Filters, sorting and bulk expand/collapse**, with a `/` keyboard shortcut for the text filter.
- 🤖 **Fix plans** generated by the OpenAI API when `OPENAI_API_KEY` is set, with a fully **deterministic fallback** when it is not.
- 📤 **Exports**: Markdown, JSON, CSV, XML, **SARIF 2.1.0** (GitHub code scanning), standalone **HTML report**, readiness **badge SVG** — plus a print-friendly report view and a copy-paste summary.
- 🕐 **Local scan history** with score deltas between runs (browser localStorage only — nothing stored server-side).
- ⚙️ **In-repo scan config** — ship a `launchguard.config.json` to suppress rules or hide low-severity noise (always disclosed in the report).
- 🌓 **Dark / light theme**, stable report **fingerprints**, scan duration and file-type breakdown.
- 🌐 **HTTP API**: `/api/rules` (catalog), `/api/health` (probe), `/api/badge?score=NN` (SVG), with **rate-limited** scan endpoints.
- 🧪 **Unit, integration and Playwright smoke tests**, wired into **GitHub Actions CI**.
- 🛡️ **Safety by construction** — scanned code is never executed, installed, imported or built; secret-like values are always redacted; uploads and downloads are never persisted.

## Quick start

Requires **Node.js 20** and **npm**.

```bash
# 1. Install dependencies
npm install

# 2. (Optional) configure environment — everything works without this
cp .env.example .env.local

# 3. Start the dev server
npm run dev
# open http://localhost:3000
```

### Try it in 10 seconds (no key required)

1. Open the app.
2. Leave the **Demo scan** tab selected and click **Run demo scan**.
3. You’ll see a readiness score, severity breakdown, filterable findings and a **Generate fix plan** button (deterministic unless you set an OpenAI key).

The demo scans a built-in, intentionally-flawed sample project. Every credential in it is fake and is redacted in the output.

### Scan a real project

- **GitHub tab:** paste a public repo URL like `https://github.com/owner/repo` (or `owner/repo`) and click **Scan repository**. LaunchGuard downloads a read-only archive — it never clones, installs or runs anything.
- **Upload ZIP tab:** drag in a `.zip` (≤ 15 MB). It is scanned in memory and never written to disk.

## Configuration

All environment variables are **optional**. See [`.env.example`](.env.example).

| Variable | Purpose | Default |
| --- | --- | --- |
| `OPENAI_API_KEY` | Enables AI-generated fix plans. Without it, a deterministic plan is produced. | _(unset)_ |
| `OPENAI_MODEL` | Model used for fix plans. | `gpt-4o-mini` |
| `GITHUB_TOKEN` | Raises the anonymous GitHub API rate limit. Public repos work without it. | _(unset)_ |

When `OPENAI_API_KEY` is set, **only already-redacted findings** are sent to OpenAI — never raw file contents or raw secrets.

### In-repo scan configuration

A scanned project may ship a `launchguard.config.json` at its root:

```json
{
  "ignoreRules": ["CFG004", "ADV001"],
  "minSeverity": "low"
}
```

- `ignoreRules` — rule ids to suppress (unknown ids are reported and ignored).
- `minSeverity` — hide findings below this severity (`"low"` hides `info`, etc.).

The config is treated as untrusted data: it is parsed defensively, and **every suppression is disclosed in the report notes** (including a count of hidden findings), so a config can never silently fake a clean report.

## HTTP API

All scan endpoints return a consistent `{ ok, ... } | { ok: false, error }` envelope. Scan, upload and fix-plan routes are **rate limited** per client (HTTP 429 + `Retry-After` when exceeded).

| Endpoint | Method | Purpose |
| --- | --- | --- |
| `/api/scan` | POST | `{ "mode": "demo" }` or `{ "mode": "github", "url": "owner/repo" }` → scan report |
| `/api/upload` | POST | `multipart/form-data` with a `file` field (`.zip`, ≤ 15 MB) → scan report |
| `/api/fix-plan` | POST | `{ report }` → prioritized remediation plan (OpenAI or deterministic) |
| `/api/rules` | GET | Machine-readable catalog of all rules |
| `/api/health` | GET | Liveness probe: status, version, rule count, uptime |
| `/api/badge?score=NN` | GET | Readiness badge as `image/svg+xml` |

## Safety model

LaunchGuard treats every scanned repository or archive as **untrusted data**:

- It **never executes, installs, imports or builds** scanned code — rules only pattern-match strings.
- ZIP uploads are **size-capped before decompression**, entry paths are **normalized and checked for traversal**, and total/per-file/count limits bound zip-bomb impact.
- GitHub scanning downloads a **read-only archive over HTTPS**; no clone, no hooks.
- **Secret-like values are redacted** everywhere they could surface (findings, exports, AI prompts).
- **Nothing is persisted**: uploads, downloaded archives and credentials are never written to disk or committed.

Out of scope by design: authentication, billing, a database and automatic deployment.
